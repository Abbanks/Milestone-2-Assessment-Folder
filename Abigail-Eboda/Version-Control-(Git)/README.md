Short explanation of what staging and committing do

Staging (git add): It allows you to pick and choose exactly which edits are ready, keeping your commits organized rather than just saving everything at once.

Committing (git commit): A commit takes everything in the staging area and wraps it into a snapshot with a unique ID and a message. Once committed, that version of the project is saved in the repository's history, allowing you to revert back to it if something breaks later.
